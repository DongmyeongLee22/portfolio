About Me


    저는 서버 및 통신에 관심이 많아 서버 백엔드 개발자가 되기 위해 공부를 하고 있습니다. 전공이 정보통신공학이라 소프트웨어 개발은 졸업작품을 준비하면서 처음 접하였습니다. 졸업 작품을 통해 PHP 기반의 웹호스팅과 안드로이드 애플리케이션을 제작하여 교내대회 최우수상과 논문 수상을 받은 경험이 있습니다. 졸업 작품을 준비하면서 대학 생활을 중 가장 의미 있고 즐거운 경험을 통해 개발자가 되기로 했습니다.
    전공과목에서 배운 데이터통신과 컴퓨터네트워크 등 통신과 관련된 기반 지식과 졸업작품을 준비하면서 경험한 데이터베이스 관리 및 서버 구축을 경험 삼아 자바 언어 기반인 Spring framework를 공부하고 있습니다. Spring의 기반이 되는 Core를 비롯하여 Spring Web MVC, Spring Security, Spring Data JPA등 스프링의 다양한 기술들을 적용하여 자그마한 프로젝트들을 만들어보며 기술들을 사용해보고 있습니다. JPA 통해 관계형 데이터베이스를 관리하고 있습니다. 연관 관계 설정 시 지연 로딩 설정에 대한 중요성을 알고 있으며 패치 조인이나 배치 쿼리를 이용하여 N+1문제를 해결하는 방법도 공부하였습니다. 비록 수많은 데이터를 관리해보지는 않았지만 어떤 식으로 쿼리를 짜고 어떤 식으로 테이블을 설계해야 트래픽이 많은 환경에서도 효율적인 작업이 이루어질 수 있는지에 대해 알아보기 위해 노력을 하고 있습니다. 아마존 웹 서비스를 이용해 EC2 서버와 RDS를 통해 MariaDB를 구축하였습니다. 그리고 Travis CI와 AWS Code Deploy, S3를 이용해 원격저장소에 PUSH만 하여도 자동으로 배포할 수 있게 하고 Nginx를 통해 무중단 배포를 가능하게 해본 경험이 있습니다.
    REST API 개발을 통해 REST DOCS를 통한 문서화 작업을 해보았고 클라이언트와 서버 간의 독립적인 개발이 가능하고 진정한 REST API 통신이 요구하는 HATEOAS와 Self-Describe Messages에 대해 이해하고 있습니다. 스프링 시큐리티를 이용하여 직접 Oauth 2.0 인증 서버, 리소스 서버를 설정하여 Token 인증을 구현해보았으며 현재는 구글, 네이버의 Oauth 2.0 서비스를 이용하여 로그인 연동하는 자그마한 프로젝트를 개발 중입니다. 그리고 백엔드만이 아닌 다양한 플랫폼을 경험해보고자 웹팩을 이용해 모듈 번들링을 통해 현재 이 페이지를 제작하였고 부트스트랩을 통해 테마를 사용하지 않고 직접 만들었습니다. 그리고 꾸준히 깃헙, Blog에 공부한 내용들을 정리하고 있습니다. 링크 주소는 사이트 하단에 있습니다.
